# Bagging Ensemble (`bagging/`)

This directory contains the **Bagging-based ensemble learning implementation** developed for the **ML Project Final** by **Madagascar Penguins**.

The main objective of this module is to implement a **Random Forest Classifier from scratch** using the bagging approach and analyze its behavior compared with boosting methods such as AdaBoost.

Bagging improves model performance by reducing variance through training multiple independent learners and combining their predictions.

---

## Overview

### What is Bagging?

Bagging (Bootstrap Aggregating) is an ensemble learning technique where multiple models are trained independently on different bootstrap samples of the original dataset.

The general workflow:

1. Create multiple bootstrap samples from the training data.
2. Train an independent model on each sample.
3. Aggregate predictions from all models.
4. Produce the final prediction.

The main benefit of bagging is reducing model variance and improving generalization.

---

# Random Forest Classifier

The main implementation in this module is:

```
RandomForestClassifier
```

Random Forest extends the bagging idea by introducing additional randomness through feature selection.

Each decision tree in the forest is trained using:

- Bootstrap sampling of training examples.
- Random feature selection during splitting.
- Independent tree construction.

The final prediction is obtained through majority voting.

---

# Implementation Details

## Bootstrap Sampling

For every tree:

- A bootstrap dataset is generated by sampling training examples with replacement.
- The bootstrap dataset has the same size as the original training set.
- Samples that are not selected are stored as Out-Of-Bag (OOB) samples.

Bootstrap sampling creates diversity between trees, which helps reduce overfitting.

---

## Decision Tree Base Learner

Each tree in the forest uses the custom implementation:

```
src/trees/decision_tree.py
```

The implementation does not use:

```python
sklearn.tree.DecisionTreeClassifier
```

as the main algorithm.

Each tree supports:

- Gini impurity
- Entropy criterion
- Maximum depth control
- Minimum samples split
- Random feature selection

---

# Random Feature Selection

Random Forest introduces additional randomness by selecting only a subset of features at each split.

Supported options:

```python
max_features="sqrt"
```

Selects √N features.

```python
max_features="log2"
```

Selects log₂(N) features.

```python
max_features=int
```

Uses a fixed number of features.

```python
max_features=None
```

Uses all available features.

Feature sub-sampling decreases correlation between trees and improves ensemble diversity.

---

# Prediction

## Class Prediction

Each decision tree produces an individual prediction.

The final Random Forest prediction is obtained using majority voting:

```
Final prediction = most common prediction among all trees
```

---

## Probability Prediction

The classifier also supports probability estimation.

Process:

1. Calculate prediction probabilities from each tree.
2. Average probabilities across all trees.
3. Return the final probability distribution.

---

# Out-of-Bag (OOB) Evaluation

The implementation supports Out-of-Bag score calculation.

Enable with:

```python
RandomForestClassifier(
    oob_score=True
)
```

OOB evaluation uses samples that were not included in the bootstrap sample of each tree.

Advantages:

- Provides an internal validation estimate.
- Does not require a separate validation dataset.
- Helps evaluate model generalization.

# Feature Importance

Feature importance is calculated by averaging the importance values of all trees in the forest.

It represents how much each feature contributes to reducing impurity during tree construction.

Access:


# Bias-Variance Perspective

Random Forest mainly improves performance by reducing variance.

A single Decision Tree:

- Can learn complex patterns.
- Has high variance.
- Can easily overfit training data.

Random Forest:

- Trains many diverse decision trees.
- Averages their predictions.
- Produces a more stable model.

This makes Random Forest more robust on unseen data.
